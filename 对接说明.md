# Inty 实时语音通话（Live Chat）对接说明

本文说明如何凭 Inty 提供的基地址与令牌，调用 **HTTP 状态接口** 与 **WebSocket 实时语音**。下行 JSON 消息类型、PCM 上行格式等以本文及 **`examples/`** 为准；联调时请以 `GET .../live-chat/status` 的返回字段与 WebSocket 实际 payload 为最终依据。

---

## 交付包内容

解压后通常包含下列条目（以运营实际交付为准）：

| 路径 | 说明 |
| --- | --- |
| `INTY_LIVE_CHAT_PARTNER_DELIVERY.md` | 主说明（与仓库内 `对接说明.md` 同源；运营 zip 内多为该文件名） |
| `inty_voice_call/` | **Android 语音 SDK 源码**（Kotlin + Ktor），与本说明**同级**；接入 Android 时整目录拷贝到您的工程 |

## 1. 联调信息（由 Inty 运营提供）

工程中的 **api_key** 即 **服务账户 JWT**，用法与常见 Bearer token 相同。请向 Inty 索取并妥善保管，**勿**写入公开仓库、勿在截图或聊天中长期明文留存。

| 字段 | 说明 | 由 Inty 填写 |
| --- | --- | --- |
| **base_url** | Inty HTTPS 基地址，**无**末尾斜杠 | （待填写） |
| **wss_base** | 与 `base_url` 同主机，将 `https` 换为 `wss` | （待填写或可自推） |
| **agent_id** | 角色 ID，用于路径 `/api/v1/live-chat/{agent_id}` | （待填写） |
| **api_key** | JWT；HTTP 与 WebSocket 均使用 `Authorization: Bearer {api_key}` | （待填写） |

**用法速查**

- **探测能力**：`GET {base_url}/api/v1/live-chat/status`，请求头 `Authorization: Bearer {api_key}`。成功时 `code` 为 200 且 `data.enabled` 为 `true`；否则 WebSocket 可能被关闭码 **4003** 拒绝。
- **实时语音**：`{wss_base}/api/v1/live-chat/{agent_id}`，握手时携带同上 Bearer（推荐）。可选查询参数示例：`speech_language_code`（如 `en-US`）、`response_language_name`（如 `English`）。也支持 `Sec-WebSocket-Protocol` 或查询参数 `token` 传 JWT；**不推荐**仅依赖 URL 中的 `token`，易进入访问日志。

## 2. HTTP：检查 Live Chat 是否可用

需已安装 `curl` 与 `jq`。在**本文件夹根目录**执行：

```bash
export INTY_BASE='https://你的基地址'
export TOKEN='你的api_key'
bash examples/http_smoke.sh
```

## 3. WebSocket：地址与鉴权（摘要）

- **路径**：`/api/v1/live-chat/{agent_id}`，接在 `wss_base` 之后。
- **鉴权**：优先使用请求头 `Authorization: Bearer <api_key>`；勿在日志或前端长期暴露完整 token。
- 连接后一般会先收到 `session_info`；上行音频为 **PCM**（常见为 `audio` 类型 JSON 帧，payload 多为 base64）。采样率须与 **`GET .../status` 响应中的 `send_sample_rate`** 一致。

## 4. Android：接入 `inty_voice_call` 模块

### 4.1 准备源码

将 **`inty_voice_call/`** 整目录拷贝到您的 Android 工程内（例如 `your-app/libraries/inty_voice_call`）。路径自定，下文以模块目录相对工程根为准。

### 4.2 注册模块

在工程 `settings.gradle.kts` 中：

```kotlin
include(":inty_voice_call")
project(":inty_voice_call").projectDir = file("libraries/inty_voice_call")
```

将 `file(...)` 改为您的实际相对路径。

在 App 模块 `build.gradle.kts` 中：

```kotlin
implementation(project(":inty_voice_call"))
```

### 4.3 构建脚本

若模块自带 `build.gradle.kts` 依赖 Inty 内部插件而无法解析，请用 **`templates/build.gradle.kts.standalone`** **替换**模块内同名文件（替换前建议备份）。再按您工程统一调整 Android Gradle Plugin、Kotlin 版本以及 `compileSdk` / `minSdk`（模板中常见为 `minSdk` 29、`compileSdk` 36）。

### 4.4 依赖与发布形态

- 依赖以模块内 `build.gradle.kts` 或 standalone 模板为准；典型包含 Ktor 3.3.x（WebSocket、OkHttp、ContentNegotiation、kotlinx-json）与 `kotlinx-serialization-json` 等。
- 本交付为 **源码模块**，**不保证**以 AAR 形式发布至 Maven Central；`group` / `version` 仅作模块元数据。

### 4.5 主要类

- `IntyVoiceCallUrls.liveChatWebSocketUrl(...)`：拼接 WebSocket URL（默认可能在查询串中带 `token`；生产环境建议在 Ktor 客户端增加 `Authorization` 头，减少 token 出现在 URL 中）。
- `IntyVoiceCallClient`、`VoiceCallWebSocketDataSource`、`CallPacket`：连接、收发与数据模型。

Kotlin 最小示例见 `examples/AndroidKtorMinimal.kt`（需拷贝到您 App 源码并补齐依赖）。

### 4.6 `demo/` 与 `assets/`（参考实现与演示录像）

**`demo/`**：摘录自 IntelliMate 主工程的 Repository / ViewModel / Ktor 工厂等，**仅供 Inty 内部或主工程对照**，**不是**客户对接步骤的一部分。对接方**无需**拷贝 `demo/` 内代码；Android 接入以 **第 4.1～4.5 节** 与 **`examples/AndroidKtorMinimal.kt`** 为准。更多说明见 **`demo/README.md`**。

**`assets/` 演示录像（`.mp4`）**：由运营在发货前放入（建议文件名 **`inty_live_chat_demo.mp4`**，也可用其他名称并在交付时说明）。用于展示**联调成功后的预期效果**，例如：

- 执行 `examples/http_smoke.sh` 后 JSON 中 `enabled`、`send_sample_rate` 等关键字段；
- （可选）WebSocket 首帧或简要语音通话界面与声音往返。

若压缩包内暂无 `.mp4`，请向 Inty 索取更新交付包或单独链接。

## 5. 非 Android 客户端

使用任意支持自定义握手头的 WebSocket 客户端；帧为 UTF-8 JSON。可参考 `examples/python_ws_minimal.py`（需 `pip install websockets`）。

在**本文件夹根目录**执行：

```bash
export INTY_WSS_BASE='wss://你的主机'
export INTY_AGENT_ID='你的agent_id'
export INTY_TOKEN='你的api_key'
python3 examples/python_ws_minimal.py
```

## 7. WebSocket 关闭码（节选）

| 关闭码 | 含义 |
| --- | --- |
| 4001 | 未认证或 token 无效 |
| 4003 | Live Chat 未开启 |
| 4010 | Agent 数量达到上限 |
| 4011 | 时长等配额限制 |

用量或订阅不足时，关闭前也可能收到 JSON 类型 `error` 帧（含 `message`、`error_code` 等），请以实际 payload 为准。

---

## 8. 安全提示

- **api_key** 等价于账号凭证；泄露后请立即联系 Inty 作废或轮换。
- 对外可称「服务账户 JWT」或「集成令牌」，避免与第三方静态 API key 混淆。
